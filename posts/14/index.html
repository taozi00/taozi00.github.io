springboot毕设
发表于 2020-05-12| 更新于 2020-05-17|Springboot
阅读量:

service调用mapper（xml）
@mapper 标记这个接口作为一个映射接口
@configure 注入到spring容器中
@ResponseBody 标注此方法转化为json
@EnableConfigurationpropeties 开启自动配置属性 启动哪个类配置属性
AOP 使用切面去拦截所有控制器的方法的请求来记录
@pointcut声明一个切入点
@Component 声明一个对象 配置bean
mvn clean install 打包成jar或者war
java -jar xx.war

使用SpringBoot 集成Spring-data-jpa,Druid连接池,thymeleaf模板实现的一个简单网上商城项目，包含后台管理
Druid 阿里巴巴的一个连接池
@bean   表面声明一个对象
@primary 表面首先使用标注的主数据源，只能声明一次

SpringBoot 内置Tomcat ，提供了自动配置，是搭建spring的脚手架，解决了复杂的 配置、混乱的依赖关系

组合注解 @springApplication相当于 @Component和 @EnableConfigurationPropeties、 @springBootConfiguration(@springConfiguration)
声明是java配置类

@propertySource 读取资源文件
@Value 注入
（”${}”） 从spring容器获取参数 、
this.属性 代表全局 区分局部和全局
@Bean 把方法的返回值注入到spring容器
@ConfigurationPropeties 通过前前缀(prefix) 读取属性配置
@Autowired 或者 构造方法 或 方法的形参 或者在方法内@ConfigurationPropeties（方法有set返回值且属性和名字一致才可以 ） 自动注入
spring容器自动装配mapper

application.yml或者application.property 全局配置
Controller类调用service，然后service 调用Dao(mapper)

@service 一般来说 加个注解给容器管理 ，标注这是一个service类

先写dao 、service 、Controller
@CrossOrigin 跨域 比如：限制转载 A域名访问B域名的数据，此时存在跨域
Dao使用通用Mapper ，extends Mapper T:对应的service Dao需要继承tk.mybatis.mapper.annotation.RegiserMapper

习惯优于配置 \ 约定大于配置

iml @service //业务注解，spring容器自动生成Bean
Controller 自动装配处理业务
@Transactional //事务注解
